## Scripts 子命令

Scripts 子命令用于管理和运行可复用的命令片段（脚本片段），适合将常用的命令行操作整理成可复用的“脚本”。  
脚本会保存在 JSON 文件中，支持占位符参数和默认工作目录。

### 基本功能
- 添加/删除/列出脚本片段
- 运行已保存的脚本，并支持传入位置参数
- 支持全局脚本文件与当前目录下的本地覆盖脚本
- 支持为脚本指定默认工作目录
- 内置危险命令检测，防止误执行高风险操作

### 数据存储说明
- 默认数据文件：`~/.fcbyk/fscripts.json`
- 当前目录下若存在 `fscripts.json`，在运行脚本时会与全局配置合并，且本地配置优先

### 基本用法
```bash
fcbyk scripts [子命令] ...
```

支持的子命令：
- `add`：添加脚本
- `run`：运行脚本
- `list`：列出脚本
- `rm`：删除脚本

### 子命令与参数说明

#### 1. 添加脚本：`scripts add`

```bash
fcbyk scripts add <名称> "<命令字符串>" [--cwd DIR] [--yes]
```

- `<名称>`：脚本名称（键），后续通过该名称调用
- `<命令字符串>`：实际要执行的命令行
- `--cwd, -C DIR`：可选，指定脚本的默认工作目录
- `--yes, -y`：可选，若脚本已存在则直接覆盖，无需交互确认

命令字符串中可以使用占位符：
- `$1`、`$2`...：会被运行时传入的参数依次替换
- `{1}`、`{2}`...：同样会被位置参数替换，适合在某些 shell 环境下避免 `$` 被提前解析

示例：
```bash
fcbyk scripts add lsproj "ls -la {1}" --cwd D:\projects
```

#### 2. 运行脚本：`scripts run`

```bash
fcbyk scripts run <名称> [ARGS ...] [--cwd DIR]
```

- `<名称>`：要运行的脚本名称
- `ARGS ...`：传给脚本命令字符串的参数，对应 `$1` / `{1}` 等占位符
- `--cwd, -C DIR`：临时覆盖工作目录（优先级高于保存时的 cwd）

行为说明：
- 会先尝试从全局 `fscripts.json` 加载脚本
- 若当前目录存在本地 `fscripts.json`，会合并并覆盖同名脚本
- 执行前会进行危险命令检测（如 `rm -rf`、强制 `git push` 等），必要时会进行二次确认

示例：
```bash
fcbyk scripts run lsproj .
fcbyk scripts run deploy_prod
```

#### 3. 列出脚本：`scripts list`

```bash
fcbyk scripts list
```

- 列出当前可用的所有脚本（包含全局和本地合并后的结果）
- 会友好地输出脚本名称及对应命令

#### 4. 删除脚本：`scripts rm`

```bash
fcbyk scripts rm <名称>
```

- 从全局数据文件中删除指定脚本
- 若名称不存在，会提示对应信息

### 注意事项
- Scripts 会直接调用系统 shell 执行命令，请在创建脚本时谨慎检查命令内容。
- 内置的危险命令检测仅做辅助提示，不能替代人工审查。
- 若脚本依赖特定工作目录，请合理使用 `--cwd` 或在命令中使用相对路径。
