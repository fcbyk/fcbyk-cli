## Pick 子命令

Pick 子命令用于开启一个抽奖 server，可用于抽奖、随机选择等场景。  

### 基本功能
- 从给定列表中随机抽取一个元素
- 支持将候选项持久化到本地数据文件中，方便重复使用
- 支持文件抽奖模式（从文件/目录中随机抽取）
- 通过 Web 界面进行可视化管理和抽奖

### 数据存储说明
- Pick 会将数据持久化到 `~/.fcbyk/data/pick_data.json`
- 数据结构中包含一个 `items` 列表用于存储候选项

### 参数说明
```bash
fcbyk pick [选项]
```

**默认行为：** 直接启动 Web 抽奖服务，默认端口 80

- `-p, --port INTEGER`  
  Web 模式下使用的端口，默认 `80`。

- `-nb, --no-browser`  
  Web 模式下，启动后不自动打开浏览器，只在终端打印访问地址。

- `-f, --files PATH`  
  启动文件抽奖 Web 服务，从指定文件或目录中随机抽取。  
  支持文件或目录路径。

- `-ap, --ask-password`  
  搭配 `--files` 使用时，提示设置管理员密码。  
  若直接回车则使用默认密码 `123456`。

### 常见用法示例

1. 启动 Web 抽奖页面（默认 80 端口）
   
```bash
fcbyk pick
```

2. 特定端口启动 Web 抽奖页面
   
```bash
fcbyk pick -p 8080
```

3. 开启文件抽奖功能
```bash
fcbyk pick -f D:\lottery\candidates.txt -p 8080 -ap
```

### 注意事项
- 文件抽奖模式下，管理员密码默认 `123456`，建议在公开场合使用时设置更安全的密码。
- 持久化数据存放在用户目录下，如需重置，可删除对应数据文件。
