## Pick 子命令

Pick 子命令用于随机抽取一个元素，可用于抽奖、随机选择等场景。  
支持从命令行临时提供列表，或从持久化列表中抽取；还支持启动 Web 抽奖页面以及文件抽奖模式。

### 基本功能
- 从给定列表中随机抽取一个元素
- 支持将候选项持久化到本地数据文件中，方便重复使用
- 支持通过 Web 页面进行可视化抽奖
- 支持文件抽奖模式（从文件/目录中随机抽取）
- 支持查看、添加、删除、清空当前持久化列表

### 数据存储说明
- Pick 会将数据持久化到 `~/.fcbyk/data/pick_data.json`
- 数据结构中包含一个 `items` 列表用于存储候选项
- 使用 `--add/--remove/--clear` 等操作会更新该文件

### 基本用法
```bash
fcbyk pick [选项] [ITEM ...]
```

在没有 `--web` / `--files` 且没有维护类操作时：
- 若命令行传入了 `ITEM ...`，直接从这些项中随机抽取
- 否则从持久化的 `items` 列表中抽取

### 参数说明

- `-c, --config`  
  显示当前数据文件中的内容并退出。  
  会输出数据文件路径以及 `items` 列表。

- `-a, --add TEXT`  
  向持久化列表中添加元素，可重复使用多次。  
  示例：`-a 张三 -a 李四`。

- `-r, --remove TEXT`  
  从持久化列表中移除指定元素，可重复使用。  
  若元素不存在，会提示对应信息。

- `--clear`  
  清空持久化列表。

- `-l, --list`  
  显示当前持久化列表内容。

- `-w, --web`  
  启动 Web 抽奖服务，在浏览器中进行抽奖。  
  会基于当前数据文件中的 `items` 列表。

- `-p, --port INTEGER`  
  Web 模式下使用的端口，默认 `80`。

- `--no-browser`  
  Web 模式下，启动后不自动打开浏览器，只在终端打印访问地址。

- `-f, --files PATH`  
  启动文件抽奖 Web 服务，从指定文件或目录中随机抽取。  
  支持文件或目录路径。

- `-pw, --password`  
  搭配 `--files` 使用时，提示设置管理员密码。  
  若直接回车则使用默认密码 `123456`。

- `ITEM ...`（位置参数）  
  若提供，代表一次性抽奖候选项列表，不会写入数据文件。

### 常见用法示例

1. 从命令行提供的列表中抽取
```bash
fcbyk pick A B C D
```

2. 向持久化列表中添加候选项
```bash
fcbyk pick --add 张三 --add 李四 --add 王五
```

3. 查看当前候选列表
```bash
fcbyk pick --list
```

4. 从持久化列表中抽取
```bash
fcbyk pick
```

5. 启动 Web 抽奖页面（使用持久化列表）
```bash
fcbyk pick --web -p 8080
```

6. 启动文件抽奖 Web 页面
```bash
fcbyk pick --files D:\lottery\candidates.txt -p 8080 --password
```

### 注意事项
- 使用 `--web` 或 `--files` 时会检测端口是否可用，端口被占用时会直接退出。
- 文件抽奖模式下，管理员密码默认 `123456`，建议在公开场合使用时设置更安全的密码。
- 持久化数据存放在用户目录下，如需重置，可删除对应数据文件。
