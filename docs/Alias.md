## 1 概述

`alias` 用于定义命令别名，将简短名称映射为可执行命令。

用户通过：

```bash
byk <name> [...args]
```

即可执行对应命令。

## 2 核心设计

本版本为重构后的统一模型：

- ❌ 移除 `run` 子命令
    
- ❌ 移除 `alias` 子命令管理层
    
- ❌ 不再区分 alias / script
    
- ✅ 统一为 **alias**
    
- ✅ 通过 **配置文件维护**
    
- ✅ 使用 **shell 执行**
    
- ✅ 支持 **参数透传 + 占位符**
    
- ✅ 支持 **全局 + 本地覆盖**
    
- ✅ 支持 **工作目录（cwd）**
    
- ✅ 支持 **安全检测（危险命令）**

## 3 查看命令列表

```bash
byk
```

- 无参数时输出：所有子命令 + 所有 alias

## 4. 配置文件

### 4.1 文件位置

全局配置：

```bash
~/.byk/alias.byk.json
```

本地配置（当前项目）：

```bash
./alias.byk.json
```

---

### 4.2 加载规则

```text
1. 加载全局配置
2. 加载本地配置（如果存在）
3. 合并（本地优先，覆盖全局）
```

---

## 5  配置格式

### 简单形式

```json
{
  "dev": "vite",
  "build": "vite build",
  "test": "jest {args}"
}
```

---

### 完整形式（支持扩展）

```json
{
  "dev": {
    "cmd": "vite"
  },
  "install": {
    "cmd": "npm i",
    "cwd": "/Users/coke/project/web"
  },
  "lint": {
    "cmd": "eslint {0} --fix"
  }
}
```

---

## 6 执行流程

```text
前置：内置子命令不匹配，走别名判断
1. 加载全局 alias
2. 加载本地 alias（覆盖）
3. 查找命令
4. 解析参数（占位符 / 透传）
5. 确定 cwd（CLI > 配置）
6. 安全检查（危险命令）
7. 执行 shell 命令
```

---

## 7 命令解析顺序

```text
1. 内置子命令（-h 等）
2. alias
3. 未命中 → 报错
```

---

## 8 参数处理规则（核心）

### 8.1 无占位符 → 自动透传

```json
"dev": "vite"
```

```bash
byk dev a b
```

→

```bash
vite a b
```

---

### 8.2 `{args}` → 全量参数

```json
"test": "jest {args}"
```

```bash
byk test a b
```

→

```bash
jest a b
```

---

### 8.3 `{0} {1}` → 精确参数

```json
"lint": "eslint {0} --fix"
```

```bash
byk lint src/index.js
```

→

```bash
eslint src/index.js --fix
```

---

### 8.4 规则总结

|条件|行为|
|---|---|
|无占位符|自动透传|
|`{args}`|替换全部参数|
|`{0}` `{1}`|精确替换|
|出现占位符|不再自动透传|

---

### 8.5 参数异常处理

|情况|行为|
|---|---|
|参数不足|报错|
|参数过多|报错（推荐）|

---

## 9. 工作目录（cwd）

### 配置定义

```json
{
  "install": {
    "cmd": "npm i",
    "cwd": "/project/web"
  }
}
```

---

### CLI 覆盖

```bash
byk install --cwd ./other-dir
```

优先级：

```text
CLI 参数 > 配置文件
```

---

## 10. 安全机制

### 危险命令检测

示例：

- `rm -rf /`
    
- `dd`
    
- 覆盖系统路径操作
    

行为：

- 默认提示或阻止执行
    

## 12. 错误处理
    
- 字段缺失 → 提示具体 key

## 13. 与旧版本差异

### 移除

|旧能力|状态|
|---|---|
|`fcbyk run`|❌ 移除|
|`fcbyk alias add`|❌ 移除|
|命令序列数组|❌ 移除|
|CLI 管理 alias|❌ 移除|

---

### 新模型

| 能力  | 新方式          |
| --- | ------------ |
| 执行  | `byk <name>` |
| 管理  | 修改 JSON      |
| 脚本  | alias        |
| 参数  | 占位符          |


## 14. 最佳实践

- 将 `alias.byk.json` 纳入版本控制
    
- 避免使用系统常见命令名
    
- 使用 `{args}` 明确参数行为
    
- 对复杂命令显式定义 cwd
